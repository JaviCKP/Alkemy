## 2024-03-24 - [O(N²) Dictionary Iteration Anti-Pattern]
**Learning:** Found an O(N²) performance bottleneck in `synthdb.graph.dependency.phase_layers`. The code was using a nested list comprehension: for each `level` in a range up to `max_phase`, it iterated over all items in a dictionary `phase_index.items()` to filter nodes belonging to that level. Since a graph can have O(N) components and O(N) levels (e.g., a straight line of dependencies), iterating the entire dictionary for every single level becomes O(N²). This is specific to Python list comprehensions over dictionaries where values represent groups, and it gets slower rapidly as the length of the dependency chain increases.
**Action:** Replace nested loops that filter dictionary keys by value with a single-pass grouping operation. Allocate an array of lists `phases` of size `max_phase + 1`, iterate over the dictionary items *once* (O(N)), and use the value as an index to distribute items into their respective lists. Finally, process (e.g., sort) each resulting list. This changes the O(N²) algorithm into an O(N) grouping + O(N log K) sorting.

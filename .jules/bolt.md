## 2025-02-28 - Optimizing Faker Determinism in Row-Based Generation
**Learning:** `synthdb` achieves determinism by re-seeding the random number generator per row using an isolated `ctx.rng`. Calling `faker.seed_instance(ctx.rng.getrandbits(64))` inside a hot loop is significantly slower (~25% slower) than directly assigning `self._faker.random = ctx.rng`. The deterministic safety remains intact because `ctx.rng` is already a dedicated, deterministically seeded `random.Random` instance for that specific row. Furthermore, caching the `getattr` provider lookup during generator initialization removes the per-row attribute lookup overhead.
**Action:** When integrating third-party random number libraries like Faker that allow custom random instance injection, prefer assigning the pre-seeded `random.Random` instance directly instead of re-seeding their internal random instance via integer generation. Also, always cache dynamic method lookups (`getattr`) before entering the generation loop.
